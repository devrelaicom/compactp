# Corpus files that produce unexpected parse errors.
#
# This file is the corpus contract. The corpus test in
# `crates/compactp_parser/tests/corpus_test.rs` compares the set of
# non-negative files that emit diagnostics against this manifest and fails
# if they differ in either direction:
#
#   * A file listed here that now parses cleanly     -> remove the line
#   * A file not listed here that emits diagnostics  -> fix the grammar
#                                                       or add it here
#
# Files under any `negative/` directory are expected to fail and are not
# tracked here. Paths are corpus-relative with forward-slash separators.
#
# Lines starting with `#` and blank lines are ignored.
#
# CATEGORIZATION SCHEMA (WS1 Phase 1 Task 8)
#
# Each entry below is annotated as:
#   path/to/file.compact    # CATEGORY[: optional note]
#
# Categories:
#   grammar-gap            -- parser missing a real production; WS1 Phase 2 work
#   upstream-bug-repro     -- reduced compiler bug; not user code
#   intentional-strictness -- deliberate parser rejection; see LS.md
#
# An uncategorized entry is treated as `unclassified` and MUST be
# resolved before WS1 Phase 1 is complete.
